localstorage

  • To store data that needs to persist even after the browser is closed or the user navigates away from the website. To store data that needs to be accessed quickly, without having to send a request to a server. To store data that should be available to the user offline
  • Sensitive information, such as passwords or personal identification numbers (PINs). Data that needs to be protected from unauthorized access or tampering
  • Strings, JSON objects, Arrays
