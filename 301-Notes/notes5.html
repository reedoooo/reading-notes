Thinking in React

  • Single Responsibility Principle (SRP) states that a component should only do one thing. It applies to components by ensuring that each component should have a single responsibility, such as rendering UI or handling user input.
  • Building a "static" version of an application means creating a version that renders the UI but does not have interactivity or state. It can help in breaking down the application into smaller components and figuring out the component hierarchy.
  • After building a static version, interactivity and state need to be added to make the application dynamic. This involves identifying which components need to have state and how to pass state down to child components.
  • Three questions to determine if something is state:
    • Does it remain unchanged over time? If so, it probably isn't state.
    • Does it remain unchanged over time? If so, it probably isn't state.
    • Can you compute it based on any other state or props in your component? If so, it isn't state.
  • State should live in the component that needs it and any components below it in the hierarchy that need to access and/or modify it.

Higher-order Functions

  • A higher-order function is a function that takes one or more functions as arguments and/or returns a function as its result.
  • The greaterThan function takes a number as its argument and returns a new function that takes another number as its argument and returns a boolean value indicating whether the second number is greater than the first. Line 2 of the function is returning a new function that takes a number as its argument and compares it to the original number passed to greaterThan.
  • Map and reduce are higher-order functions that operate on arrays. Map applies a function to each element in an array and returns a new array with the modified elements. Reduce applies a function to each element in an array and returns a single value as the result, such as a sum or concatenation of the elements.
