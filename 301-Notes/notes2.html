React LifeCycle

  • Based off the diagram, the 'constructor' happens first, followed by the 'render', and then the 'componentDidMount'.
  • The very first thing to happen in the lifecycle of React is the 'constructor' method, which is called when an instance of a component is created.
  • The correct order of the React lifecycle methods is: constructor, render, componentDidMount, React Updates, componentWillUnmount.
  • The 'componentDidMount' method is called after a component is mounted and rendered in the DOM, making it a good place to perform any necessary setup or fetch data from an API.

React State Vs Props

  • You can pass various types of data in the props, including strings, numbers, arrays, objects, functions, and even other React components.
  • The big difference between props and state is that props are passed from a parent component to a child component and cannot be modified by the child, while state is managed within a component and can be updated by the component itself.
  • We re-render our application when there is a change in the state or props of a component.
  • Examples of things that we could store in state include user input, fetched data from an API, or any other dynamic data that needs to be updated and reflected in the UI.
