REST

  • Representational State Transfer.
  • Resources, which are any kind of object, data, or service that can be accessed by the client.
  • A URI (Uniform Resource Identifier), which uniquely identifies a resource. Example: https://api.example.com/v1/products/1234
  • GET, POST, PUT, PATCH, DELETE.
  • The resources they identify, not the operations that can be performed on those resources.
  • Give an example of a good URI. https://api.example.com/v1/customers/5678/orders/1234
  • Having a 'chatty' web API means that a large number of small requests need to be made in order to perform an operation, which can impact performance. This is generally considered a bad thing.
  • 200 (OK).
  • 404 (Not Found).
  • 201 (Created).
  • 204 (No Content).
