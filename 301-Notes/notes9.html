Functional Programming Concepts

  • What is functional programming?
    • Functional programming is a programming paradigm that focuses on writing code using functions. It emphasizes the use of immutable data and avoids side effects.
  • What is a pure function and how do we know if something is a pure function?
    • A pure function is a function that has no side effects and always returns the same output for a given input. We can determine if a function is pure if it meets the following criteria:
      • It doesn't modify any external state.
      • It doesn't depend on external state.
      • It doesn't have any side effects, such as I/O operations or network requests.
      • It always returns the same output for the same input.
  • What are the benefits of a pure function?
    • Pure functions have several benefits, including:
      • They are easier to reason about and test, since they always return the same output for a given input.
      • They are less error-prone, since they don't modify external state and don't have side effects.
      • They can be easily composed together to create more complex functions.
  • What is immutability?
    • Immutability is a concept in functional programming that states that once a variable is assigned a value, it cannot be changed. Instead, new values are created when a change is made, and the original value is left unchanged. This helps to prevent unintended side effects and makes code easier to reason about.
  • What is Referential transparency?
    • Referential transparency is a property of a function that means it can be replaced with its return value without changing the behavior of the program. In other words, a function is referentially transparent if it can be substituted with its output without affecting the program's behavior.

Node JS Tutorial for Beginners #6 - Modules and require()

  • What is a module?
    • A module is a file that contains reusable code in Node.js. It can contain functions, variables, and other code that can be used in other parts of a program.
  • What does the word ‘require’ do?
    • The 'require' keyword is used to load a module into a Node.js program. It allows us to access code in other modules from the current module.
  • How do we bring another module into the file that we are working in?
    • To bring another module into the file we are working in, we use the 'require' keyword followed by the name of the module we want to load. For example, 'const myModule = require('./myModule.js');'
  • What do we have to do to make a module available?
    • To make a module available, we need to export the functions or variables we want to make available using the 'module.exports' object. This makes them accessible to other modules that load this module using the 'require' keyword. For example, 'module.exports.myFunction = myFunction;'
