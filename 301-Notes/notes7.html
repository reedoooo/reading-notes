  • Because they weren't designed to be used like that. When Fielding and his colleagues started building the web, being able to talk to any machine anywhere in the world was a primary concern. But most of the techniques developers later used to get computers to talk to each other didn't have those requirements. You just needed to talk to a small group of machines.
  • There are four HTTP methods (GET, POST, PUT, and PATCH) form the backbone of communication between clients and servers in the REST architectural style.
  • The GET method is used to retrieve a representation of a resource. It requests a representation of the specified resource, without modifying it. In other words, it is used to retrieve data from a resource on the server.
  • The POST method is used to submit an entity to the specified resource, often causing a change in state or side effectson the server. In other words, it is used to submit data to be processed to a resource on the server.
  • The PUT method is used to replace a resource on the server with the updated content provided in the request. In other words, it is used to update an existing resource on the server.
  • PATCH: The PATCH method is used to partially update a resource on the server. In other words, it is used to update only specific fields of an existing resource on the server without replacing the entire resource.
