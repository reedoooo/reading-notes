Thinking in React

  • Five steps of thinking in React: Start with a mock, break the UI into a component hierarchy, build a static version in React, identify the minimal representation of UI state, add inverse data flow.

State: A Component’s Memory

  • Reason a local variable isn’t sufficient for managing a React component: Local variables don't trigger a re-render when their values change, unlike state variables.
  • Argument to the useState hook and the two parts of its return array: The argument to useState is the initial state, and it returns an array where the first element is the current state and the second is a function to update it.
  • How Component A can access state from Component B: Component A can access Component B's state by lifting the state up to their closest common ancestor, or using context or a state management library like Redux.
