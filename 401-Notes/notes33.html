API Server Build Review

  • A query string parameter is appended to the URL with a "?" symbol and can be used to pass additional data to the API, while a path parameter forms part of the URL and is often used to identify a specific resource or resources.
  • Given the provided information, the API URL with a path id parameter would be: http://our-site.com/v3/stuff/things
  • Describing the dynamic API with an interface to a non-technical friend: It's like a vending machine. You push the right buttons (send the right request) and you get what you want (the response).

Auth Server Build Review

  • To use middleware for basic and bearer auth, you would incorporate the middleware functions into the route handling in your application. These functions would validate the credentials provided in the request and determine if the client is authorized to access the requested resources.
  • The handshake necessary for OAuth involves the application requesting authorization from the user, the user authorizing the application, the application receiving an authorization code, and then exchanging that code for an access token.
  • Explaining Role Based Access Control to a non-technical friend: It's like having different keys for different rooms in a building. Depending on your role (like being a janitor or a manager), you can access different rooms (resources).
