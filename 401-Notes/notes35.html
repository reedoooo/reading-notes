Multiple Reducers Example

  • Creating multiple reducers allows us to split the logic of our app into smaller, more manageable chunks, improving code readability and maintainability.
  • We combine multiple reducers using the combineReducers function in Redux.
  • We manage state as an immutable object to ensure that we don't directly modify the state, which helps prevent unwanted side effects and makes debugging easier.

Redux Docs: Using Combined Reducers

  • CombineReducers is a utility function to simplify the most common use case when writing Redux.
  • CombineReducers assembles the new state tree by calling every reducer with the current state and the dispatched action and combining their results.
  • You define the initial state in an app using combineReducers by providing it as the second argument to the createStore function.

Redux Docs: Combined Reducer Syntax

  • As your app becomes more complex, you would want to split your reducing functions to maintain clarity and organization of your code.
  • The combineReducers helper function turns an object whose values are different reducing functions into a single reducing function you can pass to createStore.
  • It's a popular convention to name reducers after the state slices they manage.
