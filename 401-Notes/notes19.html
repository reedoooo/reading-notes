AWS SQS vs SNS

  • Difference between SQS and SNS: SQS is a pull-based, distributed message queuing service for independent component interactions, while SNS is a push-based service that manages the delivery of messages to subscribing endpoints.
  • Use cases: SNS is typically used for email notifications, SMS messages, and the triggering of Lambda functions. SQS is used to buffer requests that require longer processing times, and to decouple components so they run independently.

AWS SNS and SQS

  • Using SQS and SNS in a "fanout" pattern: In a "fanout" pattern, a message is sent to an SNS topic, which then triggers multiple SQS queues, effectively copying the message to all queues in parallel.
  • How "push notifications" work using SNS: A "push notification" occurs when an SNS topic has a subscribing endpoint, such as a mobile device, that receives a message published to that topic.

SQS and SNS Basics

  • Usage of a Queue system like SQS in a large scale, distributed application: SQS can be used in large scale applications to manage volumes of data and traffic, to ensure system decoupling for higher robustness, and to enable message consumption at a pace appropriate to the system's processing speed.
