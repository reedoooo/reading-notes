useEffect Hook

  • Main intended use case for useEffect: To perform side effects in function components. Side effects can include data fetching, subscriptions, or manually changing the DOM.
  • Interaction of the effect’s logic with the component: The effect's logic function runs after every render, including the first render, and it runs only when the specified dependencies have changed.
  • Importance of the return value from the effect’s logic function: The return value should be a cleanup function. React will run this cleanup function when the component unmounts or before re-running the effect due to changes in dependencies.

Reflection

  • Learning goals after reading and reviewing the class README: Understanding the lifecycle of a React component and how the useEffect hook can be used to perform side effects at different stages of this lifecycle.
