Understanding Hash Tables

Data Structures and Algorithms: Notes

June 23, 2023

What is a Hash Table?

  • A hash table, also known as a hash map, is a data structure that implements an associative array abstract data type, a structure that can map keys to values.
  • Hash tables use a hash function to compute an index into an array of buckets or slots, from which the desired value can be found.
  • Ideally, the hash function will assign each key to a unique bucket, but most hash table designs employ an imperfect hash function, which might cause hash collisions where the hash function generates the same index for more than one key.

How Does a Hash Table Work?

  • When storing an item in the hash table, the programmer would provide the key and the value.
  • The hash table then applies the hash function to the key, which returns an index into the array that forms the hash table. This index determines where the value is stored.
  • When retrieving the value, the same key is used to compute the hash, which yields the array index. The value can then be found in the array at this index.
  • Collisions are handled through various techniques, such as chaining (which forms a linked list of entries with the same hash) or open addressing (which seeks to find the next empty slot when a collision occurs).
