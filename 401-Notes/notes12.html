Reading: Web Sockets

  • A Web Socket is a communication protocol that provides full-duplex communication channels over a single TCP connection.
  • The Web Socket request/response handshake involves the client sending a Web Socket handshake request to the server, and the server responds with a Web Socket handshake response. Once the connection is established, both the client and server can send and receive messages in real-time.
  • Web Sockets provide a standardized way for the server to send content to a client without first receiving a request from that client.

Reading: Socket.io Tutorial

  • The event handler io.on() in Socket.io allows us to listen for and handle events on the server-side.
  • Possible proof of life or proof that the code works as expected in Socket.io can include logging server-side events, emitting events to clients and verifying if they receive them, and checking for error messages or console output.
  • The socket.emit() function in Socket.io is used to send a specific event with data to a single client.

Reading: Socket.io vs Web Sockets

  • The difference between WebSocket and Socket.IO is that WebSocket is a communication protocol while Socket.IO is a library that provides a higher-level API for real-time communication. WebSocket is the underlying technology, and Socket.IO builds on top of it, adding additional features and compatibility for various environments.
  • You would use Socket.IO when you need real-time communication with features like event-based messaging, room management, and automatic reconnection.
  • You would use WebSockets when you require a lower-level, bare-bones implementation of real-time communication without the additional features provided by Socket.IO.

Videos: OSI Model Explained

  • The OSI model is a conceptual framework used to understand network communication, consisting of seven layers.
  • Each layer in the OSI model plays a specific role and collaborates with other layers to enable data transfer in a network.

Videos: TCP Handshakes Explained

  • TCP uses a three-way handshake process to establish a reliable connection between a client and server.
  • The client and server exchange SYN and ACK packets to synchronize sequence numbers and establish a connection.
