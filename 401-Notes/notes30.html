Choosing the State Structure

  • Five principles for structuring state: Define as little state as possible, keep state close to where it's needed, allow state to be passed easily, contain state within a component, and try to structure state to mirror the UI structure.

Passing State Deeply with Context

  • Problem Contexts aim to solve: Prop drilling - passing data through many layers of components.
  • One technique to try before useContext: Component Composition - where components are broken down into smaller components that manage their own state.
  • Hook that complements useContext for complex applications: useReducer - it helps to manage complex state logic that involves multiple sub-values or when the next state depends on the previous one.
