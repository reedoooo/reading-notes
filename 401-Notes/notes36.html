Async Actions

  • We use Redux middleware to handle side effects, like asynchronous logic, logging, and more. It provides a third-party extension point between dispatching an action and the moment it reaches the reducer.
  • The Redux Async Data Flow Diagram describes the flow of data in Redux when dealing with asynchronous actions. It typically starts with dispatching an action that requests data, followed by the async request, and then dispatching another action when the request returns.
  • We accommodate async in our Redux app by using middlewares such as Redux Thunk or Redux Saga that allows action creators to return a function instead of an action object.

Thunk Middleware

  • We need Redux Thunk middleware to allow action creators to return a function instead of an action object. This allows the action creator to perform asynchronous tasks.
  • Redux Thunk middleware allows action creators to return a function that can dispatch other actions and perform asynchronous requests.
  • Any return value from the inner thunk function will be made available as the return value of the dispatch method.
