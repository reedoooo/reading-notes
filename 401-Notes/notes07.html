Reading: Intro to JWT

  • A JSON Web Token (JWT) is a compact, URL-safe means of representing claims between two parties.
  • JSON Web Tokens should be used when a stateless and secure method of authentication or information exchange is needed.
  • Claims are expected in the payload component of a JWT.
  • JWTs alone do not guarantee security; the integrity and authenticity of the token must be verified through the signature.
  • When sending a JWT, both the sender and the receiver must know the secret key used to sign the token.
  • Concatenated content and secret should not be sent and received directly to a non-technical recruiter; secure communication channels and encryption methods should be used.
