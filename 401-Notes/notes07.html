Reading: Intro to JWT

  • A JSON Web Token (JWT) is a compact, URL-safe means of representing claims between two parties.
  • JSON Web Tokens should be used when a stateless and secure method of authentication or information exchange is needed.
  • Claims are expected in the payload component of a JWT.
  • JWTs alone do not guarantee security; the integrity and authenticity of the token must be verified through the signature.
  • When sending a JWT, both the sender and the receiver must know the secret key used to sign the token.
  • Concatenated content and secret should not be sent and received directly to a non-technical recruiter; secure communication channels and encryption methods should be used.

Videos: JWTs Explained

  • JWTs are useful for authentication and authorization purposes in web applications. They provide a secure and self-contained way to transmit information (claims) between different parties. JWTs are widely adopted, compact, and can be easily validated by the recipient without needing to consult a centralized server. They enable stateless authentication, simplify session management, and support distributed systems and APIs.
  • JWT is Compact and self-contained. Describe how this is useful to a non-technical friend. The compactness and self-contained nature of JWTs make them useful to a non-technical friend because they can be easily transmitted as a single string in various communication protocols (such as HTTP headers or URL parameters) without the need for additional server-side lookups. This means that the information contained within the JWT, such as user identity or permissions, is included directly in the token itself. As a result, the recipient can quickly and independently verify the authenticity and integrity of the token without relying on external resources or databases.
  • The three components of a JWT signature are the header, payload, and secret key.
    • The header contains the algorithm used for signature generation (e.g., HMAC, RSA) and the token type (e.g., JWT).
    • The payload contains the claims or information about the subject (e.g., user ID, permissions, expiration time).
    • Signature is created by signing the concatenation of the encoded header and payload using a secret key or private key. The signature ensures the integrity and authenticity of the token.
